Farmfam_Prototype

Farmfam_Prototype provides a Docker-based environment for preprocessing, loading, and visualizing agricultural data using Python, Superset, and MySQL. This allows for easy deployment across environments, consistent execution, and simplified data exploration without manual configuration. The project aims to automate and simplify the process of transforming raw agricultural data into meaningful insights through visualizations, making it accessible to users regardless of their technical expertise. The implementation involves Docker containers for each stage: data preprocessing, database loading, and visualization with Superset.

Handwritten Text Recognition

The Swedish_HTR project tackles the challenge of transcribing historical Swedish handwritten manuscripts. It utilizes a combination of tools: Laypa for layout analysis, Loghi HTR with transfer learning for text recognition, and Minion gadgets for auxiliary tasks. The project's goal is to achieve accurate transcriptions. Laypa identifies text areas, Loghi transcribes them using a pre-trained model adapted for Swedish, and Minion gadgets assist with data preparation and final formatting. The workflow involves text detection, segmentation, transcription, and PageXML file generation with logical reordering. The project leverages the "Labours Memory" dataset, trained and validated on a powerful platform with consistent testing procedures. It achieves a 2.56% Character Error Rate and a 44.59% Word Error Rate, demonstrating the effectiveness of the proposed method.

Hollywood Gender Dialogue Analysis

In this project, we examined a dataset containing information about 1037 Hollywood movies and their lead roles. Our objective was to predict the lead role in a movie based on various features provided in the dataset, such as revenue, year, and the number of dialogues. We initially analyzed the data to gain insights and formulated assumptions for prediction. Subsequently, we implemented Regression and Classification methods to validate our assumptions. We also introduced a naive classifier for comparison. Through Cross Validation, we assessed the accuracy of our models and identified one that yielded more precise predictions. This chosen model will be applied to a test dataset to evaluate its performance.

Big Data Analysis of Music Metadata

This project aims to analyze and process the Million Song Dataset in a scalable way using Apache Spark. It implements a standalone Spark cluster with Docker containers and analyzes song attributes like popularity. To optimize performance for different data sizes, the project investigates configurations with varying numbers of Spark workers. Results show that two workers are optimal for the tested range (10k-50k songs) and highlight potential for handling larger datasets with further resource optimization.

GitHub Stargazer Prediction Pipeline

The GitHub Star Predictor App project aims to forecast how many stars a repository will receive using machine learning. It collects data from top repositories, trains various models, and deploys the best one to predict star counts. Data integrity is ensured with RabbitMQ, and workflow orchestration uses Ansible. The project leverages Python for preprocessing, Celery and Docker for distributed processing, and Git Hooks for continuous deployment. A Flask web interface displays predictions, top repositories, and performance metrics. Finally, horizontal scaling is achieved through additional Docker workers.
